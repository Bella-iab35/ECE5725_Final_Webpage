Introduction

A lot of designers; illustrators, painters, web designers run into the same problem when they want to use a color from the real world in their digital work. They see a color they like on an object, a piece of fabric, a flower, a wall, and they want to use it in Photoshop or Procreate or whatever tool they work in. Their options aren't great. Phone cameras shift the color depending on lighting and white balance, so the eyedropper tool gives you something close but not right. Professional colorimeters exist but they cost $200 or more, which is too much for a casual user. The fallback is just guessing the color by eye, which is slow and not very accurate. We wanted to build something in between. A small handheld device that you press against an object, push a button, and get the color on screen along with all the values designers actually use, hex code, RGB, CMYK, and the closest named color. The device sends the data wirelessly to a base station, which serves a web page anyone on the same network can pull up on their phone or laptop. From the dashboard, the user can save colors to a palette and export it as an image or as an Adobe Swatch Exchange file that opens directly in design software.
