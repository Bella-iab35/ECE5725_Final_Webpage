Code

All source code is available at our project repository.

The main files are:

  • app.py: Flask backend running on the Pi 4. Includes the Bluetooth listener thread, all color math functions, palette storage, JPG and ASE export logic, and the REST API.
  • color_scan.py: Pen-side script running on the Pi Zero 2W. Handles the button input, OLED animation, sensor reading, calibration, and Bluetooth client.
  • templates/index.html: Dashboard UI.
  • static/css/style.css : Dashboard styling.
  • colorname.json: Color name database.

Work Distribution

Bella designed and 3D-printed the pen shell across three iterations using Fusion 360. She wired the breadboard prototype and the final pen assembly, including the APDS-9999 sensor, SSD1306 OLED, push button, and white LED strip. She wrote the color sensor code (minus calibration), the OLED display animation code, and the pen-side state machine that handles transitions between idle, scanning, and end states. On the dashboard side, she designed the dark-themed UI, picked and integrated the custom Londrina font, built the sidebar palette layout, the canvas drawing modal, and the edit-color modal. She also painted the final pen shell. She also coded up the website report while helping edit the sections.

Ayushi built the Pi 4 backend in Flask, including the REST API endpoints for color updates, palette management, and exports. She implemented the Bluetooth server as a daemon thread with reconnection handling. She wrote the color math library: RGB to hex, RGB to CMYK, RGB to HSL, complementary and analogous color computation, and the closest-color-name lookup using Euclidean distance against the 30,000-entry database. She implemented the JPG export using Pillow and the ASE export by writing Adobe's binary format directly with struct.pack. She added the QR code feature and fixed the IP detection logic to handle Cornell's network configuration. She also performed the sensor calibration work to derive the white-balance weights and the saturation transform. She finally drafted the PDF form of the final report.

Both team members shared integration debugging, Bluetooth pairing, network configuration, and end-to-end testing. The final demo preparation was a joint effort.
